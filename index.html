  7. What are the differences of using HTTP/2 as opposed to HTTP 1.1?
    • Http/2 is faster and more efficient. Developers have control over prioritization. Instead of the data being sent one after another, and waiting for client response, it’s sent in pieces all at once. Each data stream is assigned a weighted value, and the client assembles them together using the values. Therefore, the developer can determine which content is more important to the user and that gets loaded first. Http/2 uses multiplexing so that pieces of data don’t block others from being loaded. It uses server push(the server pushes content without being asked for it), and header compression using HPACK
